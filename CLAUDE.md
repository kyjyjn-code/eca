# SDC ECA 자동 리서치·게시 시스템 — 프로젝트 공통 규칙 (CLAUDE.md)

이 파일은 이 프로젝트에서 일하는 모든 사람·에이전트·스크립트가 공유하는 **불변의 공통 규칙**입니다.
설계 기준 문서는 `SDC_ECA_시스템_설계서_v3.md`이며, 이 문서가 유일한 설계 기준입니다.

---

## 0. SDC 공통 원칙 (모든 단계가 반드시 지킴 — 최우선)

1. **결과 보장 금지**: 합격·수상·점수를 보장하는 표현을 절대 쓰지 않는다. 별점·관련도는 "참고용"으로만 표기한다.
2. **사실 검증**: 주최·마감·링크가 확인되지 않은 활동은 게시하지 않는다.
3. **HSS / Holyseas School 언급 절대 금지**: 어떤 산출물(데이터·사이트·PR·엑셀·주석 등)에도 포함하지 않는다.
4. **지원 자격 확인**: 한국 거주·검정고시 학생이 실제로 지원 가능한 활동만 다룬다 (국적·거주 요건 확인). 지원 불가 활동은 배제한다.
5. **대상 학생**: 검정고시·대안학교 경로 포함 초·중·고, 미국 입시 준비.
6. **포스터·이미지**: 공식 주최 페이지에 공개된 것만 사용하고, 반드시 원문 출처 링크와 함께 노출한다.

> 이 6개 원칙은 3종 에이전트(researcher / curator / publisher) 지시문에도 모두 포함되어 있으며, 어떤 경우에도 우선한다.

---

## 1. 시스템 개요

매월 1일, 공신력 있는 초·중·고 대상 대회·공모전·프로그램(ECA)을 자동 수집하여 동적 마스터 데이터에 병합·정리·게시하고, 학생에게는 고정 링크 하나만 공유하면 학생이 직접 조건을 걸러 자신에게 맞는 활동을 찾게 하는 시스템. 게시 플랫폼은 **GitHub Pages**.

### 파이프라인 (매월 1일)
1. 리서치 에이전트 → `data/raw/YYYY-MM.json`
2. 큐레이터 에이전트 → `data/curated/YYYY-MM.json`
3. `scripts/merge.js` → `data/master.json` 병합 + 병합 리포트
4. `scripts/check-links.js` → 링크 생존 확인 + 게시상태 갱신 + 리포트
5. `scripts/fetch-posters.js` → 포스터 다운로드·압축 (`docs/posters/`)
6. `scripts/export-excel.js` → `records/YYYY-MM_ECA_기록.xlsx` + 아카이브 복사
7. `scripts/build.js` → `docs/index.html` + `docs/data.json` + 아카이브 스냅샷
8. 게시 에이전트 → 게시용 PR 생성 + 변경 요약 작성
9. 담당자 승인(머지) → Pages 배포

## 2. 역할 분담 (핵심 원칙)

- **판단이 필요한 일만 LLM(에이전트)이 한다**: 리서치, 큐레이션(판정·가공), PR 변경 요약 작성.
- **나머지는 결정적 스크립트가 처리한다**: 병합, 링크 검증, 포스터, 엑셀, 빌드. 매월 동일 품질 보장.
- **단계 간 연결은 항상 JSON 파일**. 한 단계가 실패해도 그 지점부터 재실행 가능.
- **데이터의 원천은 동적 마스터(`data/master.json`) 하나**. 월별 파일(`data/raw`, `data/curated`)은 수집 기록일 뿐이고, 사이트는 항상 마스터에서 만든다.
- **각 에이전트는 최소 권한만 가진다** (아래 5장).
- **스키마는 불변의 약속**이다. 임의로 칸을 늘리거나 형식을 바꾸지 않는다.

## 3. 데이터 스키마 (v3 확정본 — 불변)

`data/master.json`, `data/raw/*.json`, `data/curated/*.json`은 모두 아래 단일 스키마를 공유하는 **객체 배열**이다.
리서치는 "사실 정보" 칸만, 큐레이터는 "판정·가공" 칸을, 스크립트가 "이음" 칸을 채운다.

| 필드 | 설명 | 채우는 주체 |
|---|---|---|
| `id` | 영문 슬러그 + 연도/회차 (예: `amc-2026`, `john-locke-2026`). **수집월 포함 금지** | 리서치 |
| `활동명` | 대회·공모전·프로그램명 | 리서치 |
| `분야` | 대분류 5개 중 하나 | 리서치 |
| `과목태그` | 수학/과학/영어/인문/경제 등, 복수 가능 (배열) | 큐레이터 |
| `키워드` | 정해진 키워드 풀에서만 선택, 복수 가능 (배열) | 큐레이터 |
| `대상_원문` | "초6~고1"처럼 사람이 읽는 원문 | 리서치 |
| `대상_학년` | 필터용 정규화 값 (초/중/고, 배열) | 큐레이터 |
| `지원자격_확인` | 한국 거주·검정고시 학생 지원 가능 여부 확인 메모 (요건 원문 근거) | 리서치 |
| `주최` | 주최·주관 기관 | 리서치 |
| `신청기간_원문` | 접수 기간 원문 ("상시모집" 포함) | 리서치 |
| `근거_원문` | 마감일·대상을 확인한 공식 페이지의 해당 문구를 원문 그대로 기록 (검증 대조용) | 리서치 |
| `마감일` | 날짜 형식 (예: `2026-07-26`). 상시·미정이면 `null` 허용 | 큐레이터 |
| `주요일정` | 심사·발표 등 | 리서치 |
| `비용_원문` | "약 6~8만원" | 리서치 |
| `비용_구분` | 무료 / 소액 / 유료 | 큐레이터 |
| `핵심내용` | 한두 문장 요약 | 리서치 |
| `웹사이트` | 공식 URL (활동 상세 [링크] 버튼에 연결) | 리서치 |
| `포스터_url` | 공식 포스터 이미지 원본 주소 (있을 때만) | 리서치 |
| `포스터` | 내려받은 포스터 파일 경로 (없으면 빈칸) | 포스터 스크립트 |
| `미국입시_관련도` | 높음 / 보통 / 낮음 | 큐레이터 |
| `SDC_적합도` | ★ 1~5 (기준 기반, 보장 아님) | 큐레이터 |
| `수집월` | 최초 수집월 (예: `2026-07`) | 리서치 |
| `갱신일` | 마지막으로 재확인·갱신된 날 | 병합 스크립트 |
| `게시상태` | `게시` / `제외` (제외 시 사유 병기: 링크오류·요건미달 등) | 링크검증 스크립트·게시자 |

### id 규칙 (v3)
- 형식: `활동명영문슬러그-연도(또는 회차)`. 예: `amc-2026`, `kmo-2026`, `hmun-2026`.
- **수집월은 id에 넣지 않는다.** 같은 활동이 다음 달에 재수집되어도 동일 id가 되어 병합 시 자동 갱신 처리된다.

### 마감일 null 규칙 (v3)
- 상시모집·마감 미정 활동은 `마감일`을 `null`로 두고 `신청기간_원문`에 원문을 남긴다.
- 정렬 시 null은 맨 뒤, 카드에는 "상시" 배지로 표시한다.
- null 항목은 마감 경과로 자동 제거되지 않으므로, 매월 링크 검증에서 생존을 확인하고 링크가 죽으면 게시 제외한다.

### 대분류 5개 (확정, 고정)
1. 과학·공학·수학생명·AI
2. 국제·리더십·사회참여
3. 글쓰기·논문·인문사회
4. 스포츠·문화·봉사
5. 환경·예술·공익

- 대분류 5개는 고정. 수학·영어·경제 같은 과목 구분은 `과목태그`로 별도 관리한다.
- 한 활동이 대분류 하나에 속하면서 과목태그 여러 개를 가질 수 있다.
- 경제·비즈니스 활동(NEC, IEO 등)은 대분류 "국제·리더십·사회참여" + 과목태그 "경제·금융"으로 일관되게 잡는다.

## 4. 별점(SDC_적합도) 계산식 (확정)

합계 0~5점 → ★ 그대로 매핑, 최소 ★1 보장.

| 기준 | 배점 |
|---|---|
| 미국 입시 활용도 | 높음 2 / 보통 1 / 낮음 0 (가장 중요) |
| 접근성 (검정고시·대안경로도 지원 가능) | 가능 1 / 제한적 0 |
| 검증된 주최 (국제·학회·정부·대학) | 해당 1 / 아님 0 |
| 비용 (무료·소액) | 해당 1 / 고비용 0 |

- 미국입시_관련도 "낮음"이어도 사이트에서 빼지 않는다. 필터로만 구분해 보이게 한다.
- 첫 실행 후 별점 분포를 확인한다. 상위 활동이 4~5점에 몰리거나 접근성 항목에 변별력이 없으면 배점 보정을 검토한다(QA).

## 5. 에이전트 최소 권한 (문서 4장 근거)

| 에이전트 | 허용 | 금지 |
|---|---|---|
| researcher | 웹검색, `data/raw` 쓰기 | 게시·git·큐레이션 |
| curator | `data/raw` 읽기, `data/curated` 쓰기 | 웹검색·게시 |
| publisher | `docs` 쓰기, bash(빌드 실행), git(커밋·푸시/PR) | 웹검색·데이터 판단 |

- 에이전트에는 최대 턴 제한을 두어 폭주·비용을 방지한다.

## 6. 소스 전략 (3층 — 확정)

- **1층 핵심 씨앗** (`data/sources.json`, 거의 안 변함): 검증된 대회·프로그램 목록.
- **2층 발굴 소스**: 정부·공공 채널, 서브그리게이터(보조), 태깅 웹검색. 반드시 3층 필터 통과.
- **3층 검증 필터**: 주최·**지원자격(한국 학생 지원 가능)**·규모/역사·상업성·출처·만료. 하나라도 불충족이면 배제.

> "대학이 인정하는 활동"은 어느 대학도 공식 목록을 내지 않으므로(정성·종합 평가), 보장으로 다루지 않고 "공신력 있는 주최 + 선별성 + 국제적 인지도" 신호로만 취급한다.

## 7. 게시 방식

- LLM은 데이터·미리보기를 라이브로 바로 올리지 않는다. "게시 대기(PR)"로 만들어 알린다.
- 담당자가 요약·미리보기를 확인·승인(머지)하면 Pages에 배포된다.
- 안전장치: 리서치 결과가 비었거나 병합 후 게시 항목이 기준 건수 미만(초기 10건)이면 게시 중단. 실패 시 알림. 월별 아카이브는 항상 보존.

## 8. 실행 환경

- OS: Windows. 셸은 PowerShell(기본) 및 Bash(POSIX) 병용.
- Node.js 기반 스크립트 5종. 외부 표준 패키지 사용 허용(`sharp`, `exceljs` 등).
- 자동 실행은 GitHub Actions(UTC `0 0 1 * *` = 매월 1일 09:00 KST), 수동 실행 버튼(`workflow_dispatch`) 포함.
